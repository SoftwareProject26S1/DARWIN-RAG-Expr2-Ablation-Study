DARWIN-RAG: 대학 공지사항 기반 RAG 서비스

DARWIN-RAG는 숭실대학교 공지사항 데이터에서 사용자의 자연어 질문과 관련된 notice chunks를 검색하고, 검색된 근거를 바탕으로 답변을 생성하는 서비스다. 일반 RAG는 하나의 vector index에서 semantic similarity만으로 찾지만, 대학 공지는 category boundary가 강해 의도와 다른 notice가 상위에 올 수 있다.

사용자 질문

“장학금 신청 언제까지야?”처럼 자연어로 입력된다.

일반 RAG 한계

비슷한 표현이 학사, 장학, 채용 notice에 함께 등장한다.

설계 방향

semantic similarity와 category signal을 함께 사용한다.

서비스 출력

검색된 근거 notice를 바탕으로 답변을 생성한다.

backend / rag architecture03 / 12

서비스 구조: category 기반 RAG

대학 공지사항은 학사, 장학, 채용, 국제교류, 비교과·행사처럼 category boundary가 강하다. 그래서 “어떤 category에서 찾을 것인가”와 “그 안에서 어떤 notice chunk가 관련 있는가”를 함께 판단하도록 설계했다.

Ingestion 단계offline indexing
01공지사항 수집 → text cleaning
02chunk split → embedding 생성
03BERT category classifier로 category signal 계산
04category별 FAISS index 저장
Query 단계online retrieval
01user question embedding 생성
02BERT classifier로 P(category | query) 계산
03관련 category index 검색 후 candidate 수집
04lambda scoring → final ranking → answer generation
retrieval decision = category route selection + within-category semantic ranking
experiment 1 · bert confidence calibration04 / 12

BERT confidence를 retrieval weight로 쓰기 전에 보정했다.

일반 softmax confidence는 실제 정답 가능성보다 과도하게 높게 나올 수 있다. 이 값을 그대로 category 기반 dynamic weight로 쓰면 특정 category 영향이 과하게 반영될 수 있다.

Dataset
2000년대 record 1개를 제외하고, 2010년부터 2026년 5월 15일까지의 u-saint 공지사항으로 구성했다.

softmax(logit / T)
T > 1 → probability distribution 완만화
optimal T = 1.3919
H1. Calibration effect

ECE, NLL, Brier score가 감소하면 confidence가 actual correctness와 더 잘 정렬된다.

H2. Differentiation

true-class probability 분포 차이가 유지되면 category별 confidence 특성이 남아 있다.

experiment 2 · normalized global merge06 / 12

Exp2는 category별 후보를 전역 동일 기준으로 병합한다.

01route selection

P(c|q) ≥ theta_route인 category index만 검색한다.

02candidate expansion

각 partition에서 candidate_k_per_partition = 50으로 후보를 넓게 확보한다.

03global normalization

s_norm = (sim + 1) / 2. partition별 min-max normalization은 쓰지 않는다.

04lambda scoring

lambda_c * s_norm + (1 - lambda_c) * P_calibrated(c|q)

05deduplication

같은 document가 여러 partition에서 발견되면 max score만 사용한다.

06global ranking

모든 candidate를 하나의 final score 기준으로 정렬한다.

기존 HTML의 실험2 수치와 충돌하는 새 수치는 추가하지 않았다. 이 슬라이드는 scoring pipeline의 공식 설명만 보강한다.

conclusion / next design11 / 12

category-aware retrieval은 candidate pruning이 아니라
candidate expansion으로 재설계해야 한다.

Exp2는 P-score를 winning variant로 검증하지 못했다. 대신 낮은 성능의 원인을 분리했다. soft routing width가 좁고, ranking 전에 gold chunks가 pruned될 수 있으며, category prior는 이를 보상하기에 너무 약하다.

01

unified candidates를 먼저 구성

unified top-K와 routed category top-K를 결합해 B0의 recall 우위를 candidate pool에 남긴다.

02

lambda recalibration

validation data에서 category별 lambda를 다시 추정해 prior 효과를 분리한다.

03

gold labels를 multi-label로 구성

primary/secondary category와 multi-label query gold로 evaluation noise를 줄인다.

04

recall diagnostics 추가

ranking metrics 전에 source hit, answer-bearing hit, chunk hit, route coverage를 보고한다.

05

category prior와 routing 분리

unified candidate generation 이후에만 category prior가 signal인지 noise인지 확인한다.

06

semantic reranking

candidate expansion 뒤 cross-encoder 또는 LLM reranking으로 gold chunks를 회복한다.
